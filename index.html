Photon Tracing

I made a simple photon tracer. Which is like a path tracer, but instead of casting rays from the camera, rays are cast from the lightsource like in real life. I only implemented it that rays bounce into the scene again when they hit the surface of an object, and that already generated the image seen above, where stuff like global illumination, ambient occlusion and soft shadows are visible. But after rendering this image I kind of stopped working on it as it took about an hour to render this.

Realtime Voxel Raytracing

For a school project, I worked on a realtime voxel raytracer. I did most of the rendering code. The renderer supports having multiple grids with voxels and each grid can be rotated, scaled and translated. As optimisation, we used checkerboard rendering, which looked way worse than we thought it would.

Slamanander Engine

Slamanander Engine (yes, it's spelled like that) is a simple game engine I've worked on in my free time, it's mostly made for learning, making an actual game in it probably wouldn't be practical. At the moment it consist of an entity component system and a physically based renderer. Slamanander Engine is written in C++.

Physically Based Materials

Slamanander Engine uses physically based rendering, which was the first time for me that I worked with PBR. Each material has a metalicness map, a normal map, a roughness map and an albedo map. I used those maps instead of some more detailed material properties, because I saw that mosts artists used those maps. Or other maps, like glossiness maps, which can easily be converted to those.

Entity Component System

I wrote my own data-oriented multithreaded entity component system for this project. I don't really have much to write about, it's just an ECS, you can create systems, you can give components to entities, you can remove components from entities, you can even add and remove components from entities within the update function of a system. The dependencies between the systems are automatically generated using the components they have access to, but you can add extra dependencies between them if you want to, which can be extremely useful when you make rendering steps a system.
